Behind Story##

In this project, I worked throughout the Whole Web Development Process from Conception, Planning, Design, Development to Publishing.

[ Web Development Flow ]
- Analyzed client's needs, target audience, business operations
- Designed website layout using Sketch tool
- Worked as a Front-End developer Collaborating with a Back-End developer via Github
- Coded authentication, authorization and accounting (AAA) APIs in server side as well as participated in database design
- Tested using Jest & Cypress

Click here to check out my work !

Having a good understanding of server side code and database design helped me to see a big picture of the project. Thus I was able to communicate effectively with the Back-End developer and to debug faster.

The challenge of this project was to put the whole business on the website. Due to the default catering quantity setting, we had hours of meeting to find a way to prevent from auto creating quantity value when a client is not on the service. Since the website is interactive between admin user and client users, both cases should be considered. Our solution was to add startDate and endDate columns to users table and distinguish users into activeUsers and inActiveUsers according to them. Then only renders active users on the list for admin user and gives an access to catering request page if it is an activeUser for client users.
