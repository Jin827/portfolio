I focused on writing modular code to be able to reuse code logic and maintain the website easier.

For Reux state organization, I used normalized data structurelike the example below. So that objects can be quickly checked if they exist in Redux store.

{
  input : {
    inputValue: {...},
    tripIds = [ 1, 2, ... ],
    searchResultValues : {
     1: { id: 1, departure: YUL, destination: IC }
     2: { id: 2, ...}
    }
  }};

For instance, in order to save a searched departure value. We need to check if 'the object with Id 1' already exists in Redux store. It was simply done using an array of 'Trip Ids' and findIndex() method like below.

const isExist = result.length === 0 ? -1 : result.findIndex(r => r === id);
