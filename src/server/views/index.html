Behind Story

100% 비대면 안전 결제 중고거래 서비스를 제공하는 플랫폼으로서 자리를 잡기위해, 중고거래 최초로 모든 택배를 2천원 균일가에 이용할 수 있는 '헬로택배' 서비스를 오픈하는 프로젝트를 성공적으로 진행하였습니다.

[ 웹/모바일 웹/웹뷰 ]
- 직거래 종료
- 헬로택배 서비스 오픈

이 프로젝트를 진행하면서 기획자, 디자이너, 서버 및 앱 개발자들과 원활한 소통과 협업이 성공적인 프로젝트의 키라는 것을 다시 한번 깨달았습니다. 특히 비개발자들에게 제가 말하고자하는 것을 간단명료하게 전달하여 좀 더 효율적인 소통을 할 수 있도록 많은 노력을 하였습니다.

이 프로젝트에서 가장 힘들었던 점은 기존의 기획서 및 개발문서의 부재였습니다. 웹에서 상품등록 화면 작업을 할때, 기존 레거시 코드가 불필요한 상태값들이 많고 중복코드가 많아 가독성이 떨어지는 이슈가 있어서 아래와 같은 대대적인 리팩토링 작업을 하였습니다.

- 중복되는 코드가 많은 'rc카 상품'과 '일반 상품' 등록 화면의 통합
- 코드소스의 모듈화를 위해 로컬로 관리하던 상태를 redux를 이용하여 글로벌로 관리하도록 리팩토링

이 과정에서 기존 상품등록 로직에 대한 개발문서가 작성되어있지 않아서, 기획적인 내용을 여기저기 확인해가면서 개발해야하는 어려움으로 인해 불필요한 시간낭비를 많이 하였습니다. 개발문서 작성하고 이를 꾸준히 업데이트해가는 것이 얼마나 중요한지 다시 한번 확인할 수 있는 좋은 경험이었습니다.

웹사이트 영상 보기

데이터베이스와 서버 사이드 코드에 대한 이해도가 전체 프로젝트를 파악하는데 도움이 되어, 백앤드 개발자와 원활하게 의사소통을 할 수 있었고 버그도 더 빠르게 해결하여, 프로젝트를 좀 더 효율적으로 진행할 수 있었습니다.

웹사이트 만든다는 것은 비지니스 전체를 담아내는 것이라고 생각합니다.
이를 위해 고객의 요구사항을 최대한 수용하면서 개발의 효율성과 적합성의 발란스를 유지하는 것이 이 프로젝트에서 가장 도전적인 부분이였습니다. 프로젝트 중 고객사의 모든 회원 정보를 한눈에 보고 수정할 수 있도록 해달라는 고객의 요청이 있었습니다. 이에 저는 고심끝에 모든 회원 데이터를 테이블로 나타내고, 상대적으로 데이터가 복잡한 식수가격만 따로 클릭해서 상세정보(식수가격 변동 히스토리)를 볼 수 있도록 하였습니다. 또한, 식수가격 수정은 고객이 매년 초에 모든 회원의 식수가격을 비교, 인상하는 것을 고려하여 모든 회원의 현재 가격, 변동 가격과 적용일자를 테이블로 따로 정리하여 좀 더 쉽게 파악하고 수정할 수 있도록 하였습니다.

Behind Story##

이 프로젝트에서는 확장형 웹사이트를 만들고 유지 관리가 쉽도록, 코드 묘듈화에 중점을 두었습니다.

이의 연장선으로 효율적인 state 관리를 위해, Redux 라이브러리와 정규화된 데이터 구조를 이용하였습니다. 정규화된 데이터 구조를 적용함으로써 원하는 object를 좀 더 빠르게 찾을 수 있게 하였습니다.

{
  input : {
    inputValue: {...},
    tripIds = [ 1, 2, ... ],
    searchResultValues : {
     1: { id: 1, departure: YUL, destination: IC }
     2: { id: 2, ...}
    }
  }};

예를 들어 검색된 출발지 값을 저장하는 경우, 아이디 1을 가진 object가 redux store에 있는지 확인 후, findIndex() 를 이용하여 아래와 같이 간단하게 해결하였습니다.

const isExist = result.length === 0 ? -1 : result.findIndex(r => r === id);

* 이 웹사이트는 회사 사정으로 인해 런칭되지 않았습니다.

대학시절 많은 프로젝트에 참여하여 팀 프로젝트에 익숙한 저였지만, 이경험을 통해 처음으로 웹 개발자로서의 팀워크의 힘을 배우게 되었습니다.
