Behind Story##

이 프로젝트에서는 확장형 웹사이트를 만들고 유지 관리가 쉽도록, 코드 묘듈화에 중점을 두었습니다.

이의 연장선으로 효율적인 state 관리를 위해, Redux 라이브러리와 정규화된 데이터 구조를 이용하였습니다. 정규화된 데이터 구조를 적용함으로써 원하는 object를 좀 더 빠르게 찾을 수 있었습니다.

{
  input : {
    inputValue: {...},
    tripIds = [ 1, 2, ... ],
    searchResultValues : {
     1: { id: 1, departure: YUL, destination: IC }
     2: { id: 2, ...}
    }
  }
};

For instance, in order to save a searched departure value. Fist, We need to check if 'the object with Id 1' already exists in Redux store. It was done using an array of 'Trip Ids' and findIndex() method like below.

const isExist = result.length === 0 ? -1 : result.findIndex(r => r === id);

Behind Story##

** 캐나다 DecodeMTL에서 Full Stack Developer Bootcamp 과정의 졸업 팀 프로젝트입니다. **
총 4명의 팀원으로 구성되어있는 팀에서 저는 클라이언트 사이드 작업을 했습니다.
[ 주요 작업 ] - 로그인 인증 ( authentication ) - 프로젝트 업무 카드 - 서버 사이드와 결합 후 디버깅.

가장 흥미로웠던 것은 디버깅하는 부분이였습니다. 클라이언트 사이드와 서버 사이드를 결합 한 후 어디에서 오는지 알수 없는 버그가 여기저기에서 발생하였었습니다. 이 문제를 해결하게 위해 서버 사이드에 작업했던 팀원 한명과 제가 각각 클라이언트와 서버 사이드 코드가 켜진 두개의 모니터를 나란히 두고 라인 하나하나를 확인하면서 디버깅을 하였습니다.

대학시절 많은 프로젝트에 참여하여 팀 프로젝트에 익숙한 저였지만, 이경험을 통해 처음으로 웹 개발자로서의 팀워크의 힘을 배우게 되었습니다.
